# Biblioteca

Esta pasta guarda conhecimento derivado de ETL.

Células especializadas escrevem aqui depois de tratar fontes brutas
(transcrições, livros, anotações, materiais do expert). O conteúdo
serve de referência densa para outras células consultarem.

## O que esta pasta NÃO é

A `biblioteca/` não é uma 5ª dimensão do Ikigai.
O Ikigai tem 4 dimensões: `o-que-faco.md`, `quem-sou.md`,
`para-quem.md`, `como-faco.md`. Ver decisão D-v1.1-08.

## Organização

Cada célula que escreve aqui cria uma subpasta com o nome do tema.
Exemplo: `biblioteca/lancamentos/`, `biblioteca/copywriting/`.
A célula documenta a fonte, a data do ETL e o autor em um
`README.md` próprio na subpasta.

## Quem escreve

- Apenas células de ETL homologadas escrevem nesta pasta.
- O expert revisa antes do conteúdo virar referência consultável.
- Mudanças passam pelo Change Log da subpasta.

## Quem lê

Qualquer célula pode ler. Use o `ikigai-reader.js` apenas para as 4
dimensões; para a `biblioteca/` use leitura direta de arquivo, pois
o `ikigai-reader` é restrito ao escopo do Ikigai.
